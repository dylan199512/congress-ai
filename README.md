# Congress AI — Healthcare & AI Legislation Chatbot

A **retrieval-augmented generation (RAG)** system that lets you ask natural-language questions about how US Congress members are engaging with **healthcare** and **artificial intelligence** legislation across the 118th and 119th Congresses.

---

## How it works

```
User question
     │
     ▼
Sentence-Transformer
(encode query → float32 vector)
     │
     ├──▶ FAISS stance index ──▶ top-5 member stances
     │
     └──▶ FAISS bill index   ──▶ top-3 related bills
                  │
                  ▼
         Context block (plain text)
                  │
                  ▼
         Claude claude-3-sonnet
         (grounded answer, cites members/bills)
                  │
                  ▼
         Streamlit UI
         (answer + evidence cards)
```

### Pipeline stages

| Stage | Module | What it does |
|---|---|---|
| **Stance scraping** | `src/stances.py` | Parallelised RSS + press-page scraper for all current legislators |
| **Votes** | `src/votes.py` | GovInfo RCV packages — member votes on healthcare/AI bills |
| **Bills** | `src/bills.py` | Congress.gov paginated bill metadata + summaries |
| **Embeddings** | `src/embed.py` | SentenceTransformer encoding + FAISS index build with disk cache |
| **Retrieval** | `src/retrieval.py` | FAISS kNN search + context formatting |
| **Q&A** | `src/qa.py` | Claude RAG prompt construction + answer extraction |
| **UI** | `app.py` | Streamlit interface — query box, evidence cards, history |
| **Config** | `src/config.py` | All constants, paths, and API keys in one place |
| **Utils** | `src/utils.py` | HTTP helpers, date parsing, topic classification, validation |

## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Set API keys (recommended: environment variables)

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
export CONGRESS_API_KEY="..."
export GOVINFO_API_KEY="..."
```

Or edit `src/config.py` directly (not recommended for production).

### 3. Download the legislators dataset

```bash
curl -L https://theunitedstates.io/congress-legislators/legislators-current.csv \
     -o data/legislators-current.csv
```

### 4. Run the data pipeline (one-time)

```python
from src.stances import build_member_stances
from src.bills   import build_bills
from src.votes   import build_votes

build_member_stances()   # ~10-30 min depending on member websites
build_bills()            # ~20-60 min depending on Congress size
build_votes()            # ~5-10 min
```

Or run interactively from Colab / a notebook.

### 5. Launch the UI

```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

## Configuration

All tunables live in `src/config.py`:

| Constant | Default | Description |
|---|---|---|
| `CONGRESSES` | `[118, 119]` | Which congresses to fetch |
| `SCRAPE_MAX_WORKERS` | `8` | Parallel threads for stance scraping |
| `EMBED_MODEL_NAME` | `all-mpnet-base-v2` | SentenceTransformer model |
| `RETRIEVAL_K_STANCES` | `5` | Number of stance hits per query |
| `RETRIEVAL_K_BILLS` | `3` | Number of bill hits per query |
| `CLAUDE_MODEL` | `claude-3-sonnet-20240229` | Claude model used for answers |
| `CLAUDE_MAX_TOKENS` | `600` | Max tokens in Claude response |

## Caching

- **Embeddings** are cached in `cache/` as `.npy` files.
- **FAISS indexes** are persisted as `.faiss` binaries.
- The cache is **automatically invalidated** when the source CSV is newer than the cache file.
- Subsequent app starts skip re-encoding (typically saves 30–60 s).

## Roadmap

### Near-term improvements

- [ ] **Expand topic coverage** — add immigration, climate, defence, economy keyword lists
- [ ] **Vote alignment scoring** — compute a per-member alignment score for each topic and display it in the UI as a leaderboard
- [ ] **Historical congresses** — extend `CONGRESSES` back to the 115th for trend analysis
- [ ] **Full-text bill scraping** — fetch and embed the full bill text (not just summaries) for better semantic matching
- [ ] **Cosine similarity scores** — surface retrieval confidence alongside each evidence card

### Medium-term improvements

- [ ] **Member profile pages** — drill-down view per legislator (stances, vote history, bills sponsored)
- [ ] **Topic trend charts** — timeline of healthcare/AI legislative activity (Plotly)
- [ ] **Compare members** — side-by-side stance comparison for two legislators
- [ ] **Streamlit authentication** — restrict access to a research team

### Longer-term

- [ ] **Real-time RSS ingestion** — background scheduler refreshes stances daily without full re-scrape
- [ ] **Hybrid retrieval** — combine BM25 keyword search with dense embeddings (HyDE / RRF)
- [ ] **Cross-reference votes ↔ stances** — flag members whose voting record diverges from stated positions
- [ ] **Export to PDF/CSV** — allow researchers to download Q&A sessions with citations
- [ ] **Multi-modal** — ingest committee hearing transcripts and floor speech recordings

## Module reference

### `src/config.py`
Central registry of all constants. Import from here, never hard-code values elsewhere.

### `src/utils.py`
- `safe_get(url)` — HTTP GET with graceful error handling
- `congress_get(path)` — Congress.gov API wrapper
- `govinfo_get(url)` — GovInfo API wrapper
- `classify_topics(text)` — keyword-based topic tagger
- `parse_valid_date(s)` — normalise date strings to ISO format
- `is_valid_bioguide(bio)` — validate against legislators CSV

### `src/stances.py`
- `build_member_stances()` — parallel scrape of all ~535 legislators
- `parse_rss_for_stances(bio, url)` — parse one RSS feed
- `scrape_press_page_for_stances(bio, url)` — scrape one press page

### `src/votes.py`
- `build_votes()` — fetch RCV packages from GovInfo, filter, write CSV

### `src/bills.py`
- `build_bills()` — paginate through Congress.gov bill list, fetch details, write CSV

### `src/embed.py`
- `load_indexes()` — returns `(model, stances_df, bills_df, stance_index, bill_index)` with caching

### `src/retrieval.py`
- `retrieve(query, ...)` — returns top-k stance and bill DataFrames
- `build_context(stances, bills)` — formats evidence for LLM prompt

### `src/qa.py`
- `make_claude_client()` — creates Anthropic client
- `answer_question(query, ...)` — full RAG pipeline, returns `QAResult`
